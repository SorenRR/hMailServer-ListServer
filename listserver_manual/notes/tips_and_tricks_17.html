How To Set Up a List Where Only Certain People Can Post

Say you have a list where 100 people will be recipients of the message, but only 5 of those people are allowed to send messages to the list.  This is the real beauty of hMailServer ListServer.  The folks who can send messages via the list are called, "Posters".


You can set up the list so that your Posters do not need to know any passwords.  You do this by setting a password and adding people's email addresses to the proper Posters Lists.


•Set a password.  You really need to do this because in order to use the ListServer, you had to set the Distribution List to Public in hMailServer Administrator.  If you don't set a password, then anybody and everybody can post.  If you set a password on a list, anyone who knows that password can post to the list even if they are not a Poster.

•Now, there are three ways to control your Posters.  Note that Posters will always take the greater permissions.  If they are Global Posters, they are automatically also Domain and List Posters.

•Global Posters:  These folks can post to ANY list managed by the ListServer.  This is done in the global settings using globally_allowed_list.

•Domain Posters: These folks can post to any list in that DOMAIN.   Do this by using a Domain Posters List (simply place a text file called domainposters.txt in your domain's config folder.  It is a list of email addresses separated by carriage returns).

•List Posters: These folks can post on that list in particular.  Do this in the allowaddresses setting in the list's own config file.  Use this setting if you have someone whom you only wish to allow to post to this list and no other list.



How To Set Up a List that Allows Anyone to Post

If you want a distribution list with all the features of the list server (that is, list commands, debug, etc.), yet you want to make it so that anybody can send email to the list, just remove the password.  


How to Control What Attachments Can Be Transmitted

This is an easy one!  In the config file for your list, look for this setting:

allowed_attachments


To allow all attachments, use:

allowed_attachments=*

or

allowed_attachments=


To allow no attachments at all, use:

allowed_attachments=none


To allow only certain attachments, separate the file extensions with a hash mark (#).  All other file type attachments will be removed.

allowed_attachments=bmp#jpg##jpeg#png#tif#tiff


To allow all attachments except for certain ones, do this:

allowed_attachments=*

disallowed_attachments=exe#bat#com


If your settings do not allow any attachment, the message will be delivered anyway, but hMailServer will simply remove the attachment.  There will be no traces of the attachment, however, except in the log file where you will see that the attachment was deleted.


How to Make Sure Folks with Posting Rights Don't Accidentally Reply to the List

Consider that a certain user has posting rights (that is, he is allowed to send messages to the list).  What if he accidentally replies to his copy of the message, thinking to reply to the sender?  Since he has posting rights, that means that everyone on the list gets his reply.

To remedy this, you need to configure the ListServer to replace the "reply-to" header with the senders email list.

For this we will use a token:

addreplytoheader=%mailfrom%


How to Get the Mailer Daemons to send their failure messages to the Sender

The Return-Path in the email header can be set:

addreturnpathheader=%mailfrom%


How to Reduce Spam Ranking

If your mailfrom address is the same as your return-path addreturnpathheader address, it seems to lower your spam-ranking:

addreturnpathheader=%mailfrom%



How to Create an Automatic Unscribe Link for your Footer

To do this, you need access to a webserver.  Basically, you create a web page that has the ability to send an email.  Then you insert the link to the webpage in your mailinglist_footer.html file.  When the user clicks the link, the webpage is loaded and the user is told that they are unsubscribed.  You could insert a mailto link with the unscribe command in the hyperlink, but it will open in the user's email client rather than working automatically.


<p style="color:gray size:8pt text-style:italic">You are subscribed to the mailing list: %address%.

<br />

<a href='mailto:%address%?subject=unsubscribe'>Click here to unsubscribe</a> (Just send the mail when it is composed).

</p>
