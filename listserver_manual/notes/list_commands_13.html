List Commands and Usage


hMailServer ListServer allows for certain types of interaction with users.  They can subscribe and unsuscribe to mailing lists just by sending an email with the proper command in the subject line.  


Enable or Disable all list commands for a list with the list setting: email_subscription
If this is set to false, any message with list commands will be treated as a normal email.

If this is set to true, ANYBODY can use the available list commands, even if they are not posters nor have the password.  This means that ANYBODY can subscribe and unsubscribe.



•You can change the names of these commands in the global settings.  

•For any list you can specify which commands are available by specifying them in that list's list-level configuration using the email_subscription_allowed_cmds setting.

•To use the commands, a user - any user - sends an email with a subject line consisting of just one of the commands.  Noone ever sees any message if you add one.

•The user can be notified (or not notified) when a list command has been successful or failed by using the list settings: email_subscription_success_notification and email_subscription_success_notification

•The administrator (defined in the list's admin setting) can receive a notification of all list command usages with the list setting: email_subscription_admin_notification



Here is a list of commands that can be used:

help

sends a copy of the help message below.  This is customizable.

list

returns a list of all the members of the list

subscribe

adds sender's address to the list

unsubscribe


subscribelist

You can enter a whole list of email addresses, one on each line, and they will ALL be added to the mailing list.

unsubscribelist

Remove a whole list of email addresses from the list.

subscribeaddress

Instead of subscribing the sender, you can put ANY one email address in the body of the message, and it will be subscribed to the list

unsubscribeaddress

Unsuscribe ANY one email address from the list by placing it in the body of the message


Here is a sample message returned to a user who sent help as their subject:


Hello,


this email describes the available commands for the list test@mydomain.com


List of all possible commands:

 help:  send you this help

 list:  return the members of the list

 subscribe:  add your email address to the list

 subscribeaddress:  add the single email address given in the message body

 subscribelist:  add many email addresses, each given as single line in the message body

 unsubscribe:  delete your email address from the list

 unsubscribeaddress:  delete the single email address in the message body

 unsubscribelist:  delete many email addresses, each given as a single line in the message body

The command is given in the subject,

and any required address(es) in the body of the email.

Adding and deleting large lists may take some time -- lists above 100 members are not recommended!
