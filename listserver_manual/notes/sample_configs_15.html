Distribution List

This is a list where all of the members can collaborate.  A lot of people send messages to a lot of people.  Any member can post to the list and reply to the list.  You will notice that the muilt-subscribe commands are commented out.  This is because you should only use them with great caution, as they can easily collect and create spam.


Settings:

•password=

•allowmembers=true

•addreplytoheader=list

•allowed_attachments=none

•email_subscription=true

•addreturnpathheader=%admin%

•addsenderheader=list

•email_subscription_allowed_cmds=

•
       help
       list
       subscribe
       unsubscribe

•
       #subscribeaddress
       #unsubscribeaddress
       #subscribelist
       #unsubscribelist

•footer_file_html=mailinglistfooter.html
(see the contents of this sample footer in the sample config - it provides an unsubscription link)

Open Mailing List

In this list, a few people send a message to a lot of people.  Recipients can reply to the sender.  Anyone can join or leave the list as they like.  This is useful for an personal newsletter that folks can sign up to receive if they are interested.  This is useful to set up for use with your website, where folks can sign up for a newsletter.  Since there are only a particular number of posters (the list is controlled by admin), then you can allow the use of attachments.


Settings:

•password=

•allowmembers=true

•addreplytoheader=list

•addreturnpathheader=%admin%

•addsenderheader=list

•disallowed_attachments=exe#bat#msi#cmd#com

•email_subscription=true

•email_subscription_allowed_cmds=

•
       help
       subscribe
       unsubscribe

•footer_file_html=mailinglistfooter.html
(see the contents of this sample footer in the sample config)



Closed Mailing List

In this list, a few people send a message to a lot of people.  Only the Admin adds and removes users.  This is useful for an administration team who wants to send the same message to their large staff.  Since the staff doesn't have a choice to receive the messages or not, they don't have any subscription rights.  Since there are only a particular number of posters (the list is controlled by admin), then you can allow the use of attachments for the convenience of your posting staff.


Settings:

•password=NobodyCanGuessThis

•allowmembers=false

•allowaddresses=

•
       templetonpeck@mydomain.com
       babarracus@mydomain.com

•addreplytoheader=%mailfrom%

•addreturnpathheader=%mailfrom%

•addsenderheader=%mailfrom%

•allowed_attachments=*

•email_subscription=false

•email_subscription_allowed_cmds=

•footer_file_html=mailinglistfooter.txt


Newsletter

Here, only one person sends a message to a lot of people.  Nobody replies to anybody.  Anybody can subscribe or unsubscribe as they like. This is useful for an automated regular informational newsletter.  You will notice that the muilt-subscribe commands are commented out.  This is because you should only use them with great caution, as they can easily collect and create spam.


Settings:

•password=NobodyCanGuessThis

•allowmembers=false

•allowaddresses=

•
       hannibal@mydomain.com

•addreplytoheader=noreply@%domain%

•addreturnpathheader=%admin%

•addsenderheader=%admin%

•allowed_attachments=none

•email_subscription=true

•email_subscription_allowed_cmds=

       unsubscribe

       #unsubscribelist

•footer_file_html=newsletterfooter.txt


Spam Receptacle

This configuration is here to give you an example of what NOT to do.  If you set this up, you will soon be receiving and transmitting spam and possibly viruses to the world.  Anybody can post to this list, anybody has access to the members, and anyone can add or remove hordes of email addresses to your mailing list.  They can also send any attachments they want.


Settings:

•password=

•allowed_attachments=

•disallowed_attachments=

•allowmembers=true

•allowaddresses=

•

•addreplytoheader=list

•addreturnpathheader=list

•addsenderheader=list

•attach_recipients_to_footer=true

•email_subscription=true

•email_subscription_allowed_cmds=
