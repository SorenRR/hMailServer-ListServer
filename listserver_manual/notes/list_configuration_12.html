The List-Level config file will let you specify options for just that list without affecting the other lists in your domain.


There are descriptions in the sample config files that explain what the commands do.

Remember that the hash mark (#) creates a comment and will not be read by the script.


Don't forget to configure your domainposters.txt file!  The directions are here.


Naming the list-level settings file

The mylist_at_mydomain_com.txt is a special configuration file for that domain.  Notice its naming scheme - You replace the @ sign with _at_ and the . character with an _ character.  If you don't do it this way it just won't work.

Thus the list name

mylist@mydomain.com

will get named

mylist_at_mydomain_com

and then you add a .txt to the end for the file extension:

mylist_at_mydomain_com

Modify the Headers

addsenderheader

list | %mailfrom% | harry@mydomain.com

Add header: Sender

addreturnpathheader

list | %mailfrom% | harry@mydomain.com

Add header: Return-Path.  Setting this may help to reduce your spam ranking.  This is the address where the mailer daemons will send their message failure notices.

smtpmailfrom

list|admin|%mailfrom%| harry@mydomain.com

Modify from header for smtp session.  The default is admin.

Attachments Configuration

allowed_attachments

* | none | jpg#bmp#doc#docx#pdf

Allowed Attachments.  Default is just blank.  Leave blank or use * to specify allow all.  If you want to specify criteria, separate the file extensions with a # character.  If you specify none, all attachments will be removed without warning.  This setting is useful if you want to disallow all attachments except a few.

disallowed_attachments

exe#bat

Default is blank.  Here you may specify attachments that are specifically not allowed.  This setting is useful if you want to allow all attachments execpt a few.

allowed_attachment_size

1000 | 20000

Size allowable for each attachment in kb

allowed_attachments_total_size

20000 | 100000

Size allowable total for all attachments together in kb

List Command Settings

email_subscription

true | false

Allow subscription via email.  Send help to the list to receive further information. See List Commands for more info.

email_subscription_admin_notification

true | false

Send a copy of all issued subscription commands to the administrator.  It only notifies if they subscribe.  The other commands do not warrant any message.

email_subscription_allowed_cmds

help

list

subscribe

subscribeaddress

subscribelist

unsubscribe

unsubscribeaddress

unsubscribelist

List of allowed subscription commands.  Separate commands by a new line.  End the list with an empty line or a comment line.  Only place the commands here that you want to be available to this list.

email_subscription_success_notification

true | false

Send success notification if a subscription is made

email_subscription_failure_notification

true | false

Send failure notification if a subscription fails




** These files must be in the same directory as the list's config file.



Configuration Tokens

Configuration tokens can be used in the list-level configurations.  During the event of an inbound email, these tokens will be replaced with real values.  You can use them in the list config file for any of the settings that use text.


%admin%

address of the list administrator

%serveradmin%

address of the server administrator

%address%

address of the list, itself

%mailto%

address of the list, itself

%mailfrom%

address of the sender

%domain%

domain of the list

%date%

the date

%time%

the time, duh!

%list%

address of the list - same as %address%
