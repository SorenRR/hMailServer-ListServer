Version 2.2

by Brother Gabriel-Marie September 11, 2012

•Created this html documentation.

•Added function to load the generally allowed posters from a separate text file that will work for the entire domain - this replaced having to enter them in the script itself, which worked for all domains.

•Added RootDir so that folks can easily adjust the script to their configuration

•Updated script with Peter Hyde's additions

•Added token replacement for msg_from and msg_fromname

•Fixed call to write_notification, so that sends confirmation notices for success or delivery (this was missing in the original script).

•Fixed write_notification where If get_config_string(mailto, "success_notification") was If get_config_string(config_string, "success_notification")

•Added check to see if admin is already on the list of recipients.  If add_admin=true, and he isn't on the list, then add him.


Notes

• If email_subscription=false is set in the list's config file, then the script will not check to see if the sender is an allowed poster. If it is set to true, it should allow the address to be added to the subscription list, but should also REJECT it if it is not already a specified allowed user.  If set to true, in the main script, it seems that the email subscription overrides the check for valid posters.  Search for the following line where the function misses a step:

       write_log ("      Subscription via email activated.")

•If sender is a member, he also receives a copy - he should just get a confirmation notice; He'll get both if the confirmation is true.

•Needs to be an option so that admin gets a report of all errors and failures
